> 版本：v2.0

> 更新日期：2026-04-28

> 前端端口：7000 / API 端口：7999

---

## 一、设计原则

### 1.1 核心原则
- **移动端优先**：所有设计优先适配 360px 宽度，向上兼容平板和 PC
- **简洁克制**：家庭用户年龄跨度大，界面以清晰易用为主，避免过度设计
- **数据驱动**：热力图和价值统计是核心亮点，页面布局突出数据展示
- **操作闭环**：每个操作都有即时反馈，重要操作有二次确认

### 1.2 响应式断点
```
手机：360px - 767px（主要适配）
平板：768px - 1023px
桌面：1024px - 1920px
```

## 二、颜色系统

### 2.1 品牌色
| 变量名 | 色值 | 用途 |
|--------|------|------|
| `primary` | `#409EFF` | 主按钮、选中态、链接 |
| `primary-light` | `#E8F3FF` | 标签背景、hover 背景 |
| `primary-dark` | `#337ECC` | 按钮按下态 |

### 2.2 功能色
| 变量名 | 色值 | 用途 |
|--------|------|------|
| `success` | `#67C23A` | 成功状态、在用状态 |
| `warning` | `#E6A23C` | 闲置警告、即将过期 |
| `danger` | `#F56C6C` | 删除、已用完、错误 |
| `info` | `#909399` | 次要信息、占位符 |

### 2.3 物品状态色
| 状态 | 色值 | 说明 |
|------|------|------|
| 在用 `in_use` | `#67C23A` 绿色 | 正常使用的物品 |
| 闲置 `idle` | `#E6A23C` 橙色 | 长时间未使用的物品 |
| 已用完 `used_up` | `#909399` 灰色 | 已消耗完毕的物品 |

### 2.4 中性色
| 变量名 | 色值 | 用途 |
|--------|------|------|
| `text-primary` | `#303133` | 主要文字 |
| `text-secondary` | `#606266` | 次要文字 |
| `text-placeholder` | `#909399` | 占位符、辅助文字 |
| `bg-page` | `#F7F8FA` | 页面背景 |
| `bg-card` | `#FFFFFF` | 卡片背景 |
| `border` | `#E4E7ED` | 边框、分割线 |

### 2.5 热力图色阶
```
低频/低值：#E8F3FF（浅蓝）
中频/中值：#409EFF（品牌蓝）
高频/高值：#337ECC（深蓝）
极高频：#1E3A5F（最深蓝）

吃灰王警示：#F56C6C（红色渐深）
```

## 八、通用组件规范

### 8.1 状态徽章
```html
<!-- 在用 -->
<van-tag type="success">在使用</van-tag>

<!-- 闲置 -->
<van-tag type="warning">闲置</van-tag>

<!-- 已用完 -->
<van-tag type="info">已用完</van-tag>
```

### 8.2 空状态
```html
<van-empty image="empty" description="暂无物品" />
```
空状态提示语：
- 物品列表为空：「还没有添加任何物品，试试新增一个吧」
- 位置列表为空：「还没有添加任何位置」
- 购物清单为空：「购物清单是空的，添加一个待购吧」
- 使用记录为空：「还没有使用记录」

### 8.3 加载状态
- 列表加载：`van-list` + `loading` 属性
- 页面加载：骨架屏 `van-skeleton`
- 按钮加载：按钮内显示 `van-loading`

### 8.4 错误状态
- 网络错误：`van-empty` type="error" + 重试按钮
- 表单错误：字段下方红色提示文字

### 8.5 确认删除弹窗
```html
<van-dialog
  v-model:show="showDeleteConfirm"
  title="确认删除"
  show-cancel-button
  cancel-button-text="取消"
  confirm-button-text="确认删除"
  confirm-button-color="#F56C6C"
  @confirm="handleDelete"
>
  删除后可在"已用完"中查看历史记录
</van-dialog>
```

### 8.6 Toast 反馈
| 操作 | 反馈 |
|------|------|
| 添加成功 | `Toast.success('添加成功')` |
| 删除成功 | `Toast.success('已删除')` |
| 操作失败 | `Toast.fail('操作失败，请重试')` |
| 使用成功 | `Toast.success('使用次数 +1')` |
| 复制成功 | `Toast.success('已复制到剪贴板')` |
| 入库成功 | `Toast.success('入库成功')` |

## 九、动画与过渡

### 9.1 页面切换

## 十二、部署架构

```
┌─────────────────────────────────────────────────────┐
│              Vue 3 Frontend (Vite)                  │
│              localhost:7000 (开发)                   │
└───────────────────────┬─────────────────────────────┘
                        │ HTTP /api/v1/*
┌───────────────────────┴─────────────────────────────┐
│                 Go + Gin API                        │
│                 localhost:7999                       │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │              SQLite3 (item.db)               │   │
│  │  items / locations / categories /            │   │
│  │  usage_logs / location_logs /                │   │
│  │  shopping_records / heatmap_cache            │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### docker-compose.yaml
```yaml
services:
  api:
    build: ./server
    container_name: item-manager-api
    ports:
      - "7999:8080"
    environment:
      - DB_PATH=/data/item.db
    volumes:
      - ./data:/data
    restart: unless-stopped

  frontend:
    build: ./web
    container_name: item-manager-frontend
    ports:
      - "7000:80"
    depends_on:
      - api
    restart: unless-stopped
```
