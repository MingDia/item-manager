## 一、项目概述与目标

### 1.1 项目背景

本系统面向个人家庭用户，构建一个可持续维护的家庭物品台账系统，实现：

- 物品的精细化管理与精准定位
- 物品价值的透明化追踪
- 使用行为的可视化分析
- 购物决策的数据支撑

### 1.2 核心使用场景

**持续维护型** — 家庭物品的长期、精细化管理，而非临时性大批量整理。

### 1.3 系统定位

| 维度 | 说明 |
|------|------|
| 产品类型 | 家庭资产管理工具 |
| 部署方式 | 自建后端服务（Docker Compose 部署） |
| 客户端 | Vue Web 应用（当前）→ uniapp 小程序（后续） |
| 用户规模 | 单用户 |
| 开发阶段数据库 | SQLite3 |
| 正式部署数据库 | MongoDB / MySQL（可更换，优先级低） |
| 后端代码路径 | `/root/item-manager/server/` |
| 前端代码路径 | `/root/item-manager/web/` |
| API 端口 | 7999 |
| 前端端口 | 7000 |

### 1.4 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | Vue 3 + Vite | 当前实现 |
| 前端（未来） | uniapp | 通过 HBuilder 转换 |
| 后端 | Golang + Gin | 当前开发阶段 |
| 数据库（开发） | SQLite3 | 轻量，开发友好 |
| 数据库（正式） | MongoDB / MySQL | 可更换，优先级低 |
| 部署 | Docker Compose | 与现有 MongoDB/MySQL 共用 Podman |

## 二、双重管理模式

系统将物品划分为两类，支持差异化管理策略：

### 2.1 物品类型定义

| 类型 | 标识 | 位置变动频率 | 管理重点 | 典型物品 |
|------|------|------------|---------|---------|
| 整理型 | `flexible` | 高，频繁移动 | 快速查找、灵活记录 | 衣物、书籍、食品、药品、工具 |
| 固定型 | `fixed` | 低，一次性定位 | 位置精准、长期稳定 | 家具、大家电、硬装物品、收藏品 |

### 2.2 差异化策略

| 策略维度 | 整理型 (flexible) | 固定型 (fixed) |
|---------|------------------|---------------|
| 位置录入 | 允许快速选择历史最近位置 | 必须从空间树精确选择 |
| 位置变更记录 | 变更即通知、可选关闭 | 变更触发二次确认 |
| 闲置判断 | 阈值更宽松 | 阈值更严格 |

## 四、数据字段设计

### 4.1 物品表 `items`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | INTEGER | 是 | 主键（SQLite 自增） |
| `uuid` | TEXT(36) | 是 | 全局唯一标识（UUID v4） |
| `name` | TEXT(100) | 是 | 物品名称 |
| `category_id` | INTEGER | 是 | 所属分类 |
| `space_id` | INTEGER | 是 | 存放位置 |
| `purchase_date` | TEXT(date) | 是 | 购入日期 (YYYY-MM-DD) |
| `price` | REAL | 是 | 购入总价（元） |
| `daily_value` | REAL | 自动 | 每日价值 = price / (today - purchase_date).days |
| `total_value` | REAL | 自动 | 累计价值 = daily_value × days_since_purchase |
| `use_count` | INTEGER | 默认 0 | 被使用次数 |
| `qr_code` | TEXT(50) | 自动 | 系统码，如 `ITEM_a1b2c3d4` |
| `item_type` | TEXT | 是 | `flexible` 整理型 / `fixed` 固定型 |
| `status` | TEXT | 默认 `in_use` | `in_use`在使用 / `used_up`已用完 / `idle`闲置 |
| `created_at` | TEXT(datetime) | 自动 | 创建时间 |
| `updated_at` | TEXT(datetime) | 自动 | 更新时间 |

**自动计算逻辑：**
- `daily_value`：购入当天 = price，此后每日线性递减
- `idle` 状态：由后台任务自动标记

### 4.2 位置表 `locations`（树形结构，文件夹式）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | INTEGER | 是 | 主键 |
| `name` | TEXT(50) | 是 | 位置名称 |
| `parent_id` | INTEGER | 否 | 父级 ID，NULL 为顶级 |
| `full_path` | TEXT(200) | 自动 | 完整路径，如 `厨房/主橱柜/第二层` |
| `sort_order` | INTEGER | 默认 0 | 排序序号 |
| `created_at` | TEXT(datetime) | 自动 | 创建时间 |

**特点：** 无固定层级限制，可自由扩展（厨房/主橱柜/第二层，也可是厨房/抽屉）。

**约束：** 位置节点下若有物品挂载，禁止删除。

### 4.3 分类表 `categories`（树形结构）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | INTEGER | 是 | 主键 |
| `name` | TEXT(50) | 是 | 分类名称 |
| `parent_id` | INTEGER | 否 | 父级分类，NULL 为顶级 |
| `sort_order` | INTEGER | 默认 0 | 排序序号 |
| `created_at` | TEXT(datetime) | 自动 | 创建时间 |

**初始分类（用户提供确认后生成）：**

```
卫生间
客厅
厨房
卧室
```

其他分类由用户在系统中自行添加。

### 4.4 使用记录表 `usage_logs`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | INTEGER | 是 | 主键 |
| `item_id` | INTEGER | 是 | 物品 ID |
| `use_type` | TEXT | 是 | `manual` 手动 / `scan` 扫码 |
| `used_at` | TEXT(datetime) | 是 | 使用时间 |

### 4.5 位置变动表 `location_logs`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | INTEGER | 是 | 主键 |
| `item_id` | INTEGER | 是 | 物品 ID |
| `from_space_id` | INTEGER | 是 | 原位置 ID |
| `to_space_id` | INTEGER | 是 | 新位置 ID |
| `from_path` | TEXT(200) | 自动 | 原位置完整路径 |
| `to_path` | TEXT(200) | 自动 | 新位置完整路径 |
| `changed_at` | TEXT(datetime) | 是 | 变动时间 |

### 4.6 购物记录表 `shopping_records`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | INTEGER | 是 | 主键 |
| `item_id` | INTEGER | 否 | 对应物品（入库后关联） |
| `name` | TEXT(100) | 是 | 物品名称 |
| `category_id` | INTEGER | 否 | 分类 |
| `purchase_date` | TEXT(date) | 是 | 购买日期 |
| `price` | REAL | 是 | 购入价格 |
| `source` | TEXT(100) | 否 | 购买渠道 |
| `quantity` | INTEGER | 默认 1 | 购买数量 |
| `status` | TEXT | 默认 `pending` | `pending`待入库 / `stored`已入库 |
| `created_at` | TEXT(datetime) | 自动 | 记录时间 |

### 4.7 热力图缓存表 `heatmap_cache`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | INTEGER | 是 | 主键 |
| `date` | TEXT(date) | 是 | 统计日期 |
| `type` | TEXT | 是 | 详见下方说明 |
| `data` | TEXT(JSON) | 是 | 热力图数据 JSON |
| `computed_at` | TEXT(datetime) | 自动 | 计算时间 |

`type` 可选值：
- `usage_rank` — 使用频率排行
- `shopping_trend` — 月度购物趋势
- `value_decay` — 价值衰减曲线
- `idle_rank` — 吃灰王排行
- `category_rank` — 品类购物排行
- `location_change_rank` — 位置变动频繁度

## 五、用户操作流程

### 5.1 物品标准生命周期

```
┌──────────────┐    购买     ┌──────────────────┐   入库    ┌──────────┐   使用   ┌──────────┐
│   购物清单    │ ────────→ │ 购物记录(pending) │ ──────→ │  物品入库  │ ──────→ │ 在用物品  │
└──────────────┘            └──────────────────┘          └──────────┘          └──────────┘
                                     │                                              │
                                     │ 用完                                        │
                                     ↓                                     ┌──────────┐
                              ┌──────────────────┐                      │ 已用完   │
                              │   物品未入库      │ ←── 实际收到/拆箱 ───┘ (保留记录) │
                              └──────────────────┘
```

### 5.2 扫码入库流程

```
① 扫描二维码     →   识别物品 UUID            →   弹出物品确认卡
② 选择位置       →   展开文件夹式位置树        →   多级联动选择
③ 确认入库       →   更新 item.space_id        →   "入库成功" Toast
                    写入 location_logs
```

### 5.3 扫码/手动使用流程

```
① 扫码/点击"使用" →   识别物品 ID              →   显示物品名称卡片
② 确认使用       →   use_count +1             →   数字 +1 动画
                    写入 usage_logs              Toast "已记录"
```

### 5.4 物品位置变更流程

```
① 物品详情页        →   显示当前位置信息          →   当前位置高亮
② 点击"改位置"     →   展开位置树（新位置）       →   供选择
③ 确认变更       →   更新 item.space_id          →   位置变动日志写入
                    写入 location_logs            Toast "位置已更新"
```

### 5.5 物品用完流程

```
点击"标记用完" → status="used_up" → 物品保留在列表，状态显示"已用完"
                                  → use_count 不清零
                                  → 从"在使用"视图中移除
```

## 七、特殊场景处理规则

### 7.1 闲置物品自动标记

**规则：** 系统每日凌晨扫描所有 `status=in_use` 且 `item_type=flexible` 的物品，满足以下条件时自动标记为 `idle`：

| 条件 | 阈值 | 说明 |
|------|------|------|
| 购入天数 | `> X 天` | 可配置，默认 90 天 |
| 使用次数 | `< Y 次` | 可配置，默认 3 次 |

> `fixed` 类型物品不参与自动闲置判断。

**配置路径：** 设置 → 闲置阈值配置

### 7.2 物品删除保护

- 所有物品不允许物理删除，只能变更状态
- `used_up` 状态物品保留在数据库，可查询历史
- 大量 `used_up` 物品可归档到"历史物品"视图

### 7.3 位置节点删除限制

- 若位置节点下有物品挂载，禁止删除
- 删除前需将物品迁移至其他位置
- 树形展示时实时显示各节点物品数量

### 7.4 热力图缓存策略

**定时任务（每日凌晨 3:00 执行）：**

```
1. 遍历所有物品，计算各维度热力图数据
2. 写入 heatmap_cache 表
3. 标记闲置物品（超过阈值的 flexible 物品）
```

### 7.5 标签文本生成规则

系统为每个物品生成唯一码 `ITEM_{uuid前8位}`，展示时提供文本格式标签内容，用户复制后自行打印。

**标签文本格式（纯文本，可直接复制）：**

## 八、热力图功能详细说明

### 8.1 P0 热力图（必做）

#### 8.1.1 使用频率排行榜

- **数据来源：** `usage_logs` 按 `item_id` 聚合
- **时间维度：** 全部 / 近30天 / 近90天 / 近1年
- **展示：** Top N 列表 + 柱状图

#### 8.1.2 月度购物趋势热力图

- **数据来源：** `shopping_records` 按月聚合
- **展示：** 近12个月柱状图，颜色深浅 = 金额

#### 8.1.3 价值衰减曲线

- **计算：** 对每个物品，`daily_value = price / days_since_purchase`
- **展示：** 折线图，横轴=天数，纵轴=日均价值

#### 8.1.4 吃灰王排行榜

- **筛选：** 购入超过 30 天，使用次数 < 5 次
- **排序：** 按 `daily_value` 从低到高
- **展示：** 卡片，显示：物品名、购入日期、使用次数、日均价值、建议

### 8.2 P1 热力图（重要）

#### 8.2.1 品类购物排行热力图

- **数据来源：** `shopping_records` 按 `category_id` 聚合
- **展示：** 横向柱状图 + 饼图，各品类购物金额/次数占比

### 8.3 P2 热力图（酷炫）

#### 8.3.1 位置变动频繁度热力图

- **数据来源：** `location_logs`
- **展示：** 热力矩阵，X轴=原位置，Y轴=新位置，颜色深浅=变动次数
- **用途：** 发现不合理的位置设计
